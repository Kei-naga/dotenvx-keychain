# 設計: `run` コマンド

## 1. 目的

- [仕様書](../spec.md) の 8.2 を詳細化する。
- `run` のルート解決、鍵取得、子プロセス実行を明確にする。

## 2. 設計対象

- `--` 以降の子コマンド解釈
- プロジェクトルート探索
- ID 解決
- 鍵ソース解決
- `dotenvx run` 起動
- 終了コードとメッセージ

対象外:

- `dotenvx` 鍵生成処理
- シークレットストアの OS 別列挙実装

## 3. シーケンス

1. `--` 以降の子コマンドを取り出す。
2. 親プロセスの `DOTENV_PRIVATE_KEY` が非空か確認する。
3. 事前注入鍵がなければ、
  カレントディレクトリからプロジェクトルートを探索する。
4. 設定ファイルがあれば `id` を読む。
5. なければ対象ルートから自動 ID を生成する。
6. 事前注入鍵がなければ、ストアから秘密鍵を取得する。
7. 子プロセス環境へ `DOTENV_PRIVATE_KEY` を設定する。
8. `dotenvx run -- <command> [args...]` を起動する。
9. 子プロセスの終了結果を返す。

`run` 自身は、成功時に追加メッセージを出さず、
実行対象の出力をそのまま前面に出す。

## 4. ステップ別詳細

### 4.1 引数解釈

- `run` は `--` 以降を子コマンドとして扱う。
- `--` がなくても先頭残余引数を子コマンドとみなしてよいかは
  v1 では採用せず、使用法を単純にするため `--` を必須とする。
- `--` 以降が空なら終了コード `2` を返す。
- `--` 以降の配列は順序を保ったまま透過する。

### 4.2 ルートと ID 解決

- 親プロセスの `DOTENV_PRIVATE_KEY` が非空なら、
  `run` は事前注入鍵モードとして動作する。
- 事前注入鍵モードではルート探索、設定読込、
  自動 ID 生成を行わない。
- 事前注入鍵がない場合だけ、
  `resolveRunProject(cwd)` を使ってルート候補を決める。
- 設定ファイルが見つかった場合は、その `id` を使う。
- 設定ファイルが見つからない場合だけ、
  ルートとして確定した `cwd` から自動 ID を生成する。
- 設定ファイル破損は即座に終了コード `4` とする。

### 4.3 鍵取得

- 事前注入鍵モードでは、親プロセスの
  `DOTENV_PRIVATE_KEY` をそのまま採用する。
- 事前注入鍵がない場合だけ `secretStore.get(id)` を呼び、
  秘密鍵を取得する。
- 戻り値が `null` なら終了コード `3` を返す。
- 未検出メッセージでは `init` の再実行または
  承認済み経路からの鍵注入を促す。
- ストア障害は終了コード `4` とする。
- 事前注入鍵がある場合は、ストア利用不可だけを理由に失敗させない。
- 取得した鍵は子プロセス起動後に保持し続けない。

### 4.4 子プロセス起動

- `dotenvx` バイナリを共通ランタイム層で解決する。
- 子プロセス引数は `run -- <command> [args...]` の形で組み立てる。
- 環境変数は `process.env` をコピーし、
  `DOTENV_PRIVATE_KEY` を最後に上書きする。
- Windows では先頭コマンドの解決結果に応じて
  シェル経由を許可する。

### 4.5 成功条件

`run` が成功とみなされるのは、`dotenvx` 子プロセスが
終了コード `0` で終了した場合だけとする。

子プロセスが非 0 を返した場合は、`run` 自身の前処理が成功していても
その終了結果をそのまま返す。

## 5. 異常系

- 子コマンド未指定:
  終了コード `2`。
- 設定ファイル破損:
  終了コード `4`。
- 鍵未検出:
  終了コード `3`。
- 事前注入鍵なしでのバックエンド利用不可:
  終了コード `4`。
- `dotenvx` 起動前失敗:
  終了コード `4`。
- 未対応プラットフォーム:
  終了コード `4`。

## 6. メッセージ方針

- 鍵未検出時は `init` の再実行または
  承認済み経路からの鍵注入を 1 回だけ案内する。
- 依存関係エラーでは `dotenvx` 同梱依存の確認を促す。
- 設定ファイル破損では `.dotenvx-keychain` の修正または再生成を促す。
- CI/CD や本番環境で事前注入鍵がない場合は、
  プラットフォーム標準のシークレット注入を促す。
- 子プロセス自体の非 0 終了に対して、
  `run` 独自の追加解説は原則出さない。

例:

```text
No key found for id: my-app-v2
Run `dotenvx-keychain init` in the project root to register a key.
```

## 7. 前提

- 親プロセス環境は変更しない。
- 鍵は子プロセス系統にのみ注入する。
- 事前注入鍵がある場合は、それを最優先で使う。
- フォールバックの自動 ID は設定ファイル未発見時だけ使う。
